Markdown+Pandoc写PDF

发表于 2015-12-25   |  

为什么要用Markdown写

现在再用Word,PowerPoint什么的就太Low了…
最重要的是:
公式太难看了
公式丑的像屎
直接用Latex又太高端了不会啊…怎么办

鉴于平常使用的 Markdown 是文章神器…就用这个来写吧.

插一条内容~发现了一个新的神器…请拉到最下看…

然后就发现了一款神器-Pandoc是Haskell写出来的神器…话说知道Haskell这门艺术一般的语言这么久了…这是第一次发现这语言还有这么牛逼的作品,醉了…

Pandoc支持几乎所有的开源文字格式,支持Win/OSX/Linux..6的飞起,就是运行起来感觉稍稍有点慢(毕竟Haskell写出来的)

最重要的是:直接支持Latex的编译和内嵌公式,给Markdown写作增添了无穷威力!
Pandoc支持md->tex(can use template)->pdf,所以可以使用markdown+latex混写的写作方式,最后生成pdf…新手可以什么都不用管直接套版,老手可以避开latex中恼人的设置专注于写作

给人感觉:专注于写作就是Markdown存在的真正意义,无论用来做什么事情

再提一下PPT…Pandoc原生支持Beamer!!!只要带上参数就能直接调用Beamer自带的那些逼格报表的模板

Pandoc的Markdown语法与标准Md语法略有差别,最主要的区别是:换行用一个空行表示.其他比较牛逼的拓展就是支持表格(包括居左居中居右)

如何使用Pandoc+Beamer

正如使用Latex一样…首先要让套件支持中文
pandoc -D beamer > beamer-template.tex
先导出默认模板
在\ifxetex后面加入对中文的支持

这里容易出问题的就是\setCJKmainfont{SimHei}设置字体上.由于Win\Linux\OSX上的字体各不相同所以必须自己调整.
很操蛋的一件事情就是:生成时报错信息是几乎没有的,需要自己去.log里面看

使用命令fc-list可以打出已经安装的所有字体

如何插入公式

行内用法:
\$Tex公式\$
独立成行用法:
\$\$Tex公式\$\$

混写的实用技巧

设置页面比例

在模板文件的documentclass中(一般在第一行)加参数
例-Pandoc的默认模板:

在[]内最后加上,aspectratio=169
即可设置页面比例为16:9(默认为4:3)。
设置比例为16:10同理

设置背景图片

在模板文件中加入命令
\usebackgroundtemplate{\includegraphics[width=\paperwidth, height=\paperheight]{background.png}}

如果要对标题页单独设置背景
