CF GoodBye 2015

发表于 2015-12-31   |  

不得不说是一场悲伤的比赛…套版界的耻辱
Div2基本没有<1800的了…然后我还没上过1800…

A. New Year and Days

给你某一周内的某一天或者某一月内的某一天,问你一年内有多少个这样的日子

坑:2016是闰年,闰年的2月是29天…

B. New Year and Old Property

例如2015的2进制表示11111011111,只有一个0(不含前导0)
给出[l,r],(l,r<=10^18)问有多少个这样的数在区间内

枚举出所有数判断即可,并不需要数位DP

C. New Year and Domino

给出一个$500 \times 500$的棋盘,上面有若干障碍物.给出10w个询问,询问棋盘中某个区域能有多少种放下$1 \times 2$的方块

两种情况:横着放或者竖着放,分别统计这两种情况,每行做前缀和.每次询问按行扫一遍即可

D. New Year and Ancient Prophecy

昨天就到此为止了…
给出一个n<=5000的整数,求有多少种分割方案(取模),可以将他分隔成严格递增的多个数
例如

  1. 20152016

    • “20152016” = “20152016”
    • “20152016” = “20” + “152016”
    • “20152016” = “201” + “52016”
    • “20152016” = “2015” + “2016”

  2. 123434

    • “123434” = “123434” (maybe the given sequence is just one big number)
    • “123434” = “1” + “23434”
    • “123434” = “12” + “3434”
    • “123434” = “123” + “434”
    • “123434” = “1” + “23” + “434”
    • “123434” = “1” + “2” + “3434”
    • “123434” = “1” + “2” + “3” + “434”
    • “123434” = “1” + “2” + “3” + “4” + “34

第一个想法

f[i][j]表示末位为第i位,最后一个数从第j位开始的方案数量.不太好转移

第二个想法

f[i][j]表示最后一段为(i,j)的方案数.这种表示方法有一种优势:若转移到f[l][r]+=f[i][j],那么f[l][r->n]都要+=f[i][j],所以可以在转移的时候降低复杂度.方法是将f[]当做前缀和序列,每次碰到ij的时候f[i][j]+=f[i][j-1],由于转移的原因(总是向序列的右边转移),正确性显而易见.

下面就是解决转移位置的问题了,即我们要求i和j+1两个位置开始的LCP来进行比较.这时候有两种方案进行选择:

  1. 后缀数组
  2. exKMP

后缀数组跑完以后就有了任意位置之间lcp,所以可以在任意f的计算顺序下使用(先i后j或者先j后i),但…后缀数组没用过心里没底…

exKMP是跑出主串每一个位置与模式串的LCP.所以必须先i后j的计算f.对每一个i跑一次exKMP然后O(1)回答出每一个j的转移位置

套了个版…坑点就来了…ext[]是从0开始的…a[]从1开始…
于是…T_T…套版界的耻辱…
