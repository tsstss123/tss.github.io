发表于 2016-01-16   |  

20/33
一开始就卡B,后来C写太蠢了还没过.差分约束不会.GG
留了很多坑.一定要补上.

A

题意:一个傻逼玩石头剪刀布.他的策略是一个类似递归的方法,很傻比.求某一次比赛你应该出什么能战胜它.

按照它构造的策略递归一下就可以了.

B

题意:2*N个方格要涂黑.规则是:第一个随便涂,之后的只能涂任意一个黑块的8连块上白块.

一直被卡,一直想推出来.惨啊.结束以后问他们…要么是爆搜一发然后找规律,发现两项的商是等差数列.要么是OEIS出来的.

后来问了Sifu和柱神.大概可以这么DP: f[i][j]表示宽度为i的区间(或者说前i列/某i列)，已经染了j个格子,f[i][j]=f[i][j-1]*(2*i-(j-1))+2*f[i-1][j-1](有误,为f[i][j]=f[i][j-1]*(2*i-(j-1))+4*f[i-1][j-1]) 这是怎么推出来的呢? 对于只有一行的情况f[i][j]=f[i+1][j]+f[i][j-1] 可以发现符合j-i=k的情况下f[i][j]的值是一定的.这个思想可以沿用下去...只要知道了区间长度与染色格子数量,就可以求出还有多少个格子可以染色. 这个转移方法就有点像计算卡塔兰数.OEIS上的通项公式也存在卡塔兰数

韩爷在讲题的时候说,可以通过一行的情况推出两行的公式.非常神奇,大家都听得一脸懵逼.这个留坑!

填坑

用柱神给的方程恰好和答案少了2^(n-1)倍.为什么”新开一列放一格”有四种情况呢?
原因是:i实际应该表示的是某连续的i列.如果表示成前i列的话,就无法转移(你无法得到填了2-i列第一列最后填的这种情况)(或者说,2*i-(j-1)个只涂了一个的列中,默认了第1列已经涂色).所以要加上左右两种情况.

另:柱神一开始没有看到8连块以为是4连块,就变成了另一道题…
dp[i][j][k][l][m]表示[i,j]这个区间，每列都有染色，并且总共染了k个格子，并且i列染了l个格子（1或2），j列染了m个格子（1或2）
一样可以发现对于k=j-i是一样的
优化成dp[i][j][k][l]，O(N^2)解决

C

题意:N个点N条边.N<=10w.Q<=1w个询问两点间的最短距离.

写蠢了.DFS+手工栈处理成了一个环吊着一坨树,每棵树跑LCA.询问分类处理.(当时NOI我就这么瞎比搞过,死得惨的经历还没忘…)还迷之WA

实际上找到环的时候删掉那条边,树上跑DFS+LCA.设删掉的边(u,v)询问dist(a,b).答案就是min{d(a,b),d(a,u)+w(u,v)+d(v,b),d(a,v)+w(v,u)+d(u,b)}

Sifu又教了我一种新的找环姿势,更加优雅:像拓扑排序一样,不断地删掉度为1的点,剩下的就是那个环.

留坑

D

题意:一个酒鬼从一个起点开始,在无向图上走.走进某一条路的概率和那条路的权值成正比.你可以在他的起点上去掉一条连着的边(或者不去除任何边).问最大的期望.

不会搞..沈爷的方程:dp[u[sz[u]-w(dp[v]+1))/sz[u]-w

F

题意:你有一个三层,18条边的那种,大三角套着小三角的那种三角形.两个人博弈:轮流选择一条边.如果某人选的边构成了一个小三角形,他就”获得”这个小三角形.给出了一个初始情况,求问最多能拿到多少小三角形.

和昨天那个博弈是一样的原理.18条边直接状压去搞.有一个细节:加一条边有可能能拿到两个小三角形

留坑

G

题意:给出n,m<=1000, 要你求是否存在一组a(1,2,…,n),b(1,2,…,m),存在YES不存在就NO.给不超过1w个条件..ai+bj <=c 或者 >=c

对a + b >= c改成a - c >= -b..就是a向-b连一条-c的边
a + b <= c 改成-b + c >= a..就是a是-b向a连一条c的边

当场没搞出来.

H

题意:给出了一种很傻比的字符串加密方式,要你求出最小长度的密码,没有就-1.

找到最长的那个字符串,从小到大枚举密码的长度就行.

I

题意:按照栈的顺序给出了有关sinx,cosx,tanx,+,-,*,这几种运算的表达式.求问这个表达式是否恒等于0 瞎B暴力就行了.
